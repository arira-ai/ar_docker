# Demo Dangling Docker images

### What is a Dangling Image?

A dangling image is a Docker image that is no longer tagged and is not referenced by any container. 

* It usually appears as `<none>:<none>` when you list your Docker images. 

* They take up disk space and are usually safe to delete.

---

### Prerequisites

Make sure you have Docker installed and running on your system.

## Step 1: Create a Simple Dockerfile

First, let's create a very basic Dockerfile. In an empty directory, create a file named `Dockerfile` and add the following content:

```dockerfile
FROM alpine:latest
CMD ["echo", "Hello, Version 1"]
```

## Step 2: Build the First Image

Open your terminal in the same directory and build an image from this Dockerfile. We will tag it as `dangling-demo:latest`.

```bash
docker build -t dangling-demo:latest .

```

Let's verify the image was created:

```bash
docker images dangling-demo

```

**Expected Output:**
You should see your image listed with the `latest` tag.

```text
REPOSITORY       TAG       IMAGE ID       CREATED         SIZE
dangling-demo    latest    abc123def456   5 seconds ago   7.34MB

```

### Step 3: Modify the Dockerfile

Now, let's update the code. Open the `Dockerfile` again and change the `CMD` instruction slightly to represent a new version of your app:

```dockerfile
FROM alpine:latest
CMD ["echo", "Hello, Version 2 - I have updated!"]

```

### Step 4: Rebuild with the Same Tag

Rebuild the image using the **exact same tag** (`dangling-demo:latest`):

```bash
docker build -t dangling-demo:latest .

```

### Step 5: Observe the Dangling Image

Let's look at our Docker images now.

```bash
docker images

```

**Expected Output:**

```text
REPOSITORY       TAG       IMAGE ID       CREATED          SIZE
dangling-demo    latest    xyz987uvw654   10 seconds ago   7.34MB
<none>           <none>    abc123def456   2 minutes ago    7.34MB

```

**What happened?**
Because you assigned the tag `dangling-demo:latest` to the *new* build, Docker stripped that tag away from the *old* build. The original image (ID: `abc123def456`) still exists on your hard drive, but it no longer has a name or tag. **This is your dangling image.**

---

### Step 6: How to Find Dangling Images

If you have a lot of images, looking for `<none>` tags manually is tedious. Docker has a built-in filter to find them:

```bash
docker images -f "dangling=true"

```

*Note: This will only list images that are truly untagged and unreferenced. If an untagged image is currently being used by a running or stopped container, it might not be considered "dangling" for deletion purposes until the container is removed.*

### Step 7: How to Clean Up

Dangling images just waste disk space. The safest and standard way to remove all dangling images at once is to use the `prune` command:

```bash
docker image prune

```

Docker will warn you:

```text
WARNING! This will remove all dangling images.
Are you sure you want to continue? [y/N]

```

Type **`y`** and press Enter.

**Expected Output:**

```text
Deleted Images:
deleted: sha256:abc123def456...

Total reclaimed space: 0B

```

*(The reclaimed space might show as 0B if the layers are still shared with the `alpine:latest` base image, but the specific `<none>` image layer is gone).*

Run `docker images` one last time, and you will see the `<none>:<none>` image has disappeared!
